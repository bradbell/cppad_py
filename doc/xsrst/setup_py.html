Syntax¶

python3 setup.py install --prefix= prefix

External Requirements¶

  1. bash

  2. python version 3

  3. numpy

  4. cmake

  5. swig

  6. c++

  7. git

Install Errors¶

If you get an error message during the install procedure above, or the one below, see install_error. This will only install the release version. Installing a debug version is discussed below in the instructions for downloading and building from the source code.

Download¶

Use the following command to download the current version of cppad_py:

        git clone https://github.com/bradbell/cppad_py.git top_srcdir

Top Source Directory¶

The directory you choose for top_srcdir is referred to as your top_srcdir directory.

Configure¶

Before running setup_py or bin/get_cppad.sh , you should check and possibly change the settings in bin/get_cppad.sh .

Get cppad¶

The next step is to get a copy of cppad using get_cppad_sh. If you want to use the Laplace Approximation of Mixed Effects Models class, you will have to set include_mixed to true and use bin/get_cppad_mixed.sh to install cppad and other non-standard requirements.

prefix¶

The value prefix is the prefix where cppad_py will be installed. This is determined by the command line argument to setup.py.

libdir¶

The value libdir is the suffix (after the prefix) where the libraries will be installed. This can be determined by running the command:

bin/libdir.py ; echo

LD_LIBRARY_PATH¶

Make sure the following directory is in your LD_LIBRARY_PATH (in mac OS this path is called DYLD_LIBRARY_PATH):

        prefix/libdir

PKG_CONFIG_PATH¶

Make sure the following directory is in your PKG_CONFIG_PATH:

        prefix/libdir/ pkgconfig

PYTHONPATH¶

Make sure the following directory is in your PYTHONPATH:

        prefix/libdir/ python3. minor /site-packages

where minor is the minor version corresponding to python3.

Install¶

Use the following command to build and install cppad_py:

        python3 setup.py install --prefix= prefix

This will install cppad_py in the directory

        prefix/libdir/ python3. minor /site-packages/cppad_py

The following command can be used to test the installed version of cppad_py:

        python3 example/python/check_all.py

The following packages are additional requirements to execute check_all.py: scipy matplotlib.

Test¶

These steps are optional and used to test that cppad_py works on your system before installing it. They do not yet work on mac OS.

build_local.py¶

Build a local copy of the Python cppad_py module using the command:

        bin/build_local.py

This will create the top_srcdir/ cppad_py directory.

python¶

The next step is to test the cppad_py on your system by executing the following commands starting in top_srcdir :

        python3 example/python/check_all.py

This test will use the local copy of top_srcdir/ cppad_py if it is available. You can test the installed version, if it is available, by removing the directory top_srcdir/ cppad_py and re-executing the command above.

c++¶

After bin/build_local.py has run, you can also test the cppad_py c++ interface cpp_lib on your system by executing the following commands starting in top_srcdir :

        cd build
        make check_example

import¶

If you are in the top_srcdir directory, you should be able to import cppad_py using the following commands:

        python3
        import cppad_py
        quit ()

Install Using Pip¶

There is an old version of cppad_py available using pip . To install for you entire system use:

        pip install -i https://test.pypi.org/simple/ cppad_py
