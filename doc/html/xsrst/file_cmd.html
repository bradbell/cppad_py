Syntax¶

{xsrst_file
        start
        stop
}

{xsrst_file
        start
        stop
        file_name
}

Purpose¶

A code block, from any where in any file, is included by the command above at the beginning of a line.

White Space¶

Leading and trailing white space is not included in start, stop or file_name. The new line character separates these tokens.

file_name¶

If file_name is not in the syntax, the code block is in the current input file. Otherwise, the code block is in file_name. This file name is relative to the directory where xsrst.py is executed; i.e., the top directory for this git repository. This may seem verbose, but it makes it easier to write scripts that move files and automatically change references to them.

start¶

The code block starts with the line following the occurence of the text start in file_name. If this is the same as the file containing the command, the text start will not match any text in the command. There must be one and only one occurence of start in file_name, not counting the command itself when the files are the same.

stop¶

The code block ends with the line before the occurence of the text start in file_name. If this is the same as the file containing the command, the text stop will not match any text in the command. There must be one and only one occurence of stop in file_name, not counting the command itself when the files are the same.

Spell Checking¶

Spell checking is not done for these code blocks.
